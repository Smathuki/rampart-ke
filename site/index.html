The problem

Every Kenyan organisation wants to plug AI into support chats, intake forms and triage. But doing so ships citizens' KRA PINs, National IDs, M-Pesa records, health details and names to servers outside Kenya — in direct tension with the Data Protection Act, 2019 and the ODPC's growing enforcement. Today's options are "don't use AI" or "trust the vendor." Neither is acceptable.

How it works

Two layers run on the user's device, in ~100 ms, with no server round-trip — and keep working offline after first load.

Detect locally

Deterministic Kenyan recognizers + a small on-device model find the PII in the browser (WebAssembly/WebGPU).

Redact before egress

Each value becomes a placeholder like [KRA_PIN_1]. Only placeholdered text is ever sent to an AI or server.

Restore on display

Real values are rehydrated locally for the user. The plain values never leave the device.

Built for Kenya

Localized recognizers, not a Western tool with a Kenyan sticker. Coarse geography (counties) is kept for analytics; the identifying detail is redacted.

Proof of compliance, built in

Generate an ODPC-style compliance report — how many PII instances were redacted, by category, by period. Crucially, the audit log itself holds counts only, no personal data, so keeping it is not a new liability. Evidence for regulators and boards, on demand.

Hosted API for systems that can't run client-side

The on-device library is the recommended path — there, data never leaves the browser. For legacy or back-office systems that can't run WebAssembly, this API does the redaction server-side. Host it in-country (e.g. a Kenyan data centre) so citizen data still stays in Kenya.

⚠️ Server-side means the text does reach this service. Deploy it inside your own infrastructure / a Kenyan data centre — never a foreign cloud region — to preserve data sovereignty.
Click “Redact via API”. First call warms the model (a few seconds), then it's fast.
# Same call from your backend: curl -s http://localhost:8787/api/redact \ -H 'content-type: application/json' \ -d '{"text":"My KRA PIN is A012345678Z, call 0712345678"}' # → {"redacted":"My KRA PIN is [KRA_PIN_1], call [KE_PHONE_1]","redactions":2,"labels":{"KRA_PIN":1,"KE_PHONE":1}}

A Digital Public Good

Open source (CC BY 4.0), built on National Design Studio's open Rampart model, running on open web/ONNX standards with no lock-in. It gives Global-South citizens — starting with Kenya — a way to use modern AI without surrendering their identity data to foreign servers. Privacy and do-no-harm by design; works offline for low-bandwidth settings.

Who it's for

Government & eCitizen

Redact citizen data on-device before it touches a ministry system or an external AI. Available as an annual enterprise licence.

Fintech & health-tech

Ship AI features without shipping regulated data (M-Pesa, IDs, health records) off-device.

Any Kenyan SaaS

A free npm package to drop into your front-end today, plus a hosted API for legacy back-ends.

rampart-ke — a first-line technical control that materially reduces data-transfer exposure (not, by itself, a complete DPA compliance programme).
